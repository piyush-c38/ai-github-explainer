# Visualization Architecture

## Overview

The visualization features will provide users with a graphical representation of the repository's structure, dependencies, and code flow.

## Tech Stack

*   **Graph Library:** React Flow
*   **Diagramming:** Mermaid

## Visualization Types

1.  **Folder Structure Graph:** A tree-like graph that shows the folder and file structure of the repository.
2.  **Dependency Graph:** A graph that shows the dependencies between different files and modules.
3.  **Component Relationship Graph (for React):** A graph that shows how React components are related to each other.
4.  **Request/Data Flow Graph:** A graph that visualizes the flow of data through the application (e.g., from an API endpoint to a database).
5.  **Architecture Overview Graph:** A high-level diagram of the project architecture, generated using Mermaid.

## Implementation

*   **React Flow:** We will use React Flow to create the interactive graphs. The data for the graphs will be generated by the backend during the analysis process. The frontend will fetch this data and use it to render the graphs.
*   **Mermaid:** The backend will generate Mermaid markup for the architecture overview diagram. The frontend will use the Mermaid library to render the diagram.

## Component Structure (`apps/frontend/src/components/graphs`)

*   **`FolderStructureGraph.tsx`**: Renders the folder structure.
*   **`DependencyGraph.tsx`**: Renders the dependency graph.
*   **`ComponentRelationshipGraph.tsx`**: Renders the React component relationships.
*   **`DataFlowGraph.tsx`**: Renders the data flow.
*   **`ArchitectureDiagram.tsx`**: Renders the Mermaid diagram.

Each of these components will be responsible for fetching its data and rendering the corresponding visualization.
